Phase 3: Key ag-kit Prompts (Full Library)
These are the exact prompts to use at each build step. Copy-paste into your ag-kit environment.
Hero Section
Build HeroSection:
Use the @frontend-specialist agent.

Build HeroSection.jsx. Full-screen viewport, cream/rose warm gradient background. Centered bakery name in Playfair Display serif, large and elegant. Accept props for 4 cake images (cake1-4.webp) and 2 topping images (berry.webp, mint.webp).

Each cake: initial position off-screen corner, spring animation to final position on mount (stiffness: 80, damping: 20), static CSS rotate per cake (-15, 8, -8, 12 degrees), infinite Y-axis float (amplitude 20px, 3s period, easeInOut, repeatType mirror).

Toppings: float at 1.4x speed of cakes for depth illusion. cake2.webp must have layoutId='featured-hero-cake'. Stagger mount delays: 0, 0.1, 0.2, 0.3s.

Fully responsive — cakes scale down and reposition on mobile. Respect prefers-reduced-motion.
Featured Section
Build FeaturedSection:
Use the @frontend-specialist agent.

Build FeaturedSection.jsx with Framer Motion layoutId shared transition. Grid of 4 slots: 3 featured cake cards + 1 EmptyPlate placeholder (slot index 3) with dashed border and subtle plate image.

cake2.webp card must have layoutId='featured-hero-cake' matching HeroSection. Wrap both sections in a single LayoutGroup in App.jsx. Spring transition: stiffness 200, damping 30, duration 0.6s. whileInView on section container triggers the transition when entering viewport.

Output FeaturedSection component AND the exact changes needed in HeroSection.
Menu Section
Build MenuSection:
Use the @frontend-specialist agent.

Build MenuSection.jsx. CSS grid of MenuItem cards (name, price, description, image). Each card: whileInView={{ opacity: 1, y: 0 }}, initial={{ opacity: 0, y: 40 }}, viewport={{ once: true, amount: 0.2 }}. Stagger via transition={{ delay: index * 0.1 }}.

Accept menuItems prop: { id, name, price, description, image }. Tailwind for grid and card styling. Cards should look premium: rounded-2xl, white background, subtle drop shadow.
Testimonials Section
Build TestimonialsSection:
Use the @frontend-specialist agent.

Build TestimonialsSection.jsx with 3 review cards. NO Framer Motion — Tailwind CSS only. Hover state: -translate-y-3, shadow-2xl, transition-all duration-300 ease-out. Cards: rounded-2xl, white bg, subtle border. Accept testimonials prop: { id, reviewerName, quote, stars }. Stars as filled/empty circles or simple text display. Responsive: row on desktop, column on mobile.
Phase 4: Debug & Enhancement Prompts
Debug layoutId transition:
/debug
The layoutId transition between HeroSection cake2.webp and its FeaturedSection card is janky. The cake jumps to position rather than smoothly animating. Diagnose: 1) Is LayoutGroup correctly wrapping both sections? 2) Are there conflicting CSS position values? 3) Is AnimatePresence interfering? Fix and output corrected code.
Enhance for performance:
/enhance
Review the full cake shop app for: (1) Animation performance — only transform and opacity should be animated, no layout-triggering properties. Add will-change: transform to all floating elements. (2) Mobile — cakes should not overflow viewport on screens < 768px. (3) prefers-reduced-motion — disable all infinite float animations. Wrap in a custom useReducedMotion hook. (4) Image lazy loading — all non-hero images should have loading='lazy'.
Deploy to Vercel:
/deploy
Deploy this Vite + React static site to Vercel. Build command: npm run build. Output directory: dist. Ensure all WebP image assets in /public are included in the build output. Generate vercel.json if any routing config is needed.
